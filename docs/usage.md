# Usage

`abtfexport` fetches resources from the Airbyte API and writes them as Terraform configuration. This guide covers configuration, authentication, the full set of export flags, and the files that get generated.

For migrating connections (with sync state) between Airbyte instances, see the [State Migration guide](state-migration.md).

## Basic Usage

```bash
# Export all Airbyte resources (sources, destinations, and connections) to Terraform
abtfexport --api-url https://api.airbyte.com --client-id "your-client-id" --client-secret "your-client-secret"

# Export from a specific workspace
abtfexport --workspace "workspace-id" --client-id "..." --client-secret "..."

# Export a specific connection only
abtfexport --connection-id "connection-id" --client-id "..." --client-secret "..."

# Split resources into separate files
abtfexport --split --client-id "..." --client-secret "..."
```

**Important**: If you use Self-Managed Enterprise or an Open Source deployment, your URL will need to include `/api/public` at the end. For example, `https://airbyte.contoso.com/api/public`.

## Configuration

You can configure the tool using any of the following (CLI flags take precedence over environment variables, which take precedence over the config file):

1. **Configuration file** (`~/.abtfexport.yaml`):
```yaml
# Example configuration file for abtfexport
api:
  url: "https://api.airbyte.com"  # Can also be set via AIRBYTE_API_URL environment variable
  client_id: "your_client_id"     # Can also be set via AIRBYTE_API_CLIENT_ID environment variable
  client_secret: "your_client_secret"  # Can also be set via AIRBYTE_API_CLIENT_SECRET environment variable
```

2. **Environment variables**:
```bash
export AIRBYTE_API_URL="https://api.airbyte.com"
export AIRBYTE_API_CLIENT_ID="your-airbyte-client-id"
export AIRBYTE_API_CLIENT_SECRET="your-airbyte-client-secret"
```

3. **Command-line flags** (see `abtfexport --help` for all options):
```bash
abtfexport --api-url https://api.airbyte.com --client-id "..." --client-secret "..."
```

## Getting an Airbyte Access Token

To use this tool, you'll need an Airbyte client ID and secret:

1. Log into your Airbyte account
2. Go to Settings → Account → Applications
3. Create a new application

See the [Airbyte API documentation](https://docs.airbyte.com/using-airbyte/configuring-api-access) for more details.

## Flag Reference

Authentication and workspace flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--api-url` | `https://api.airbyte.com` | Base URL of the Airbyte API |
| `--client-id` | | Airbyte API client ID for authentication |
| `--client-secret` | | Airbyte API client secret for authentication |
| `--workspace` | | Airbyte workspace ID to filter resources |
| `--config` | `$HOME/.abtfexport.yaml` | Config file location |

Export behavior flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--output-dir`, `-d` | `.` | Directory to write Terraform files |
| `--split` | `false` | Write each resource type to a separate file |
| `--connection-id` | | Export only a specific connection and its associated source and destination |
| `--migrate` | `true` | When `true`, skip generating `import` blocks. See the note below. |
| `--provider-version` | (latest) | Specific Airbyte provider version to use |
| `--skip-version-check` | `false` | Skip fetching the latest provider version and use a fallback |
| `--include-variables` | `true` | Include `variables.tf` content inside `airbyte.tf` (single-file mode only) |
| `--separate-variables` | `false` | Generate a separate `variables.tf` file instead of inlining it |
| `--skip-providers` | `false` | Skip generating `providers.tf` |

Migration-specific flags (`--migrate-connection-state`, `--no-ignore-config-drift`) are documented in the [State Migration guide](state-migration.md).

> **Note on `--migrate`:** this flag defaults to `true`, which means import blocks are **not** generated by default. Set `--migrate=false` to emit `import` blocks for adopting existing Airbyte resources into Terraform state.

## Generated Files

By default, all resources are written to a single `airbyte.tf`, alongside:

- `variables.tf` — variable definitions (inlined into `airbyte.tf` unless `--separate-variables` is set)
- `terraform.tfvars.example` — template for the secret values referenced as variables
- `providers.tf` — the Airbyte provider configuration (omitted with `--skip-providers`)

With `--split`, resources are written to separate files instead of a single `airbyte.tf`:

- `sources.tf`
- `destinations.tf`
- `connections.tf`

Secrets in source/destination configuration are replaced with Terraform variable references, so fill in the real values in `terraform.tfvars` (copy from `terraform.tfvars.example`) before applying.
